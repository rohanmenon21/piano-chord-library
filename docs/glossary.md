# Glossary

## App terms

**App**  
The website itself.

**Song**  
A saved chord sheet with a title, artist, key, and content.

**Setlist**  
A named collection of songs, arranged in order.

**Preview**  
The reading view that shows chords and lyrics as the user will see them.

**Performance mode**  
A larger live-use view for playing from the screen.

**Transpose**  
To move a song up or down by a number of semitones so it fits a different key.

**Autosave**  
Saving automatically after a short pause while the user types.

**Undo delete**  
A short window after deletion where the song can be restored.

## Backend terms

**Supabase**  
The hosted service used here for authentication and database storage.

**Database**  
A structured place to store songs, setlists, and profiles.

**RLS**  
Row Level Security. A rule system that ensures users can only access their own rows of data.

**Serverless function**  
A tiny backend function that runs only when called, instead of a full server that stays on all the time.

**Environment variable**  
A setting supplied from outside the code, usually for secrets or deployment-specific values.

**Anon key**  
A public Supabase key that identifies the app. It is not the same as a private password.

## Local browser terms

**localStorage**  
Small browser storage that keeps simple preferences even after the page is closed.

**Mock mode**  
A demo mode that uses fake in-memory Supabase data instead of the real backend.

**Smoke test**  
A small test that checks the app starts and the key screens still work.

## Musical terms

**Chord voicing**  
A specific arrangement of the notes that make up a chord.

**Semitone**  
The smallest step between two adjacent notes on a piano.

**Capo key / original key**  
The key the song starts in before transposition.
